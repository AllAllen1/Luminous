# Luminous
Видеохостинг 

Для разработки видеохостинга Luminous с функционалом загрузки и скачивания видео, на начальном этапе (Pre-project / Initiation) необходимо подготовить набор базовых артефактов. Эти документы помогут структурировать видение проекта и послужат ТЗ для дальнейшей разработки.

На основе вашего запроса, вот список и описание ключевых выходных артефактов:

1. Vision & Scope (Видение и границы проекта)
Это главный документ, который определяет «зачем» мы это делаем и «что» именно будет входить в первую версию (MVP).

Видение: Создание высокопроизводительной платформы Luminous для обмена видеоконтентом с акцентом на свободу доступа (возможность скачивания).

Границы (In-scope): Система хранения видео, плеер, личный кабинет, функционал асинхронной загрузки и генерация ссылок на скачивание в разных форматах.

Вне границ (Out-of-scope): Например, на первом этапе можно исключить стриминг в реальном времени или сложную систему монетизации.

2. Бизнес-требования (BRD)
Определяют цели проекта с точки зрения бизнеса.

Цель: Привлечь X пользователей за первый квартал за счет уникальной функции легального и удобного скачивания контента.

Ключевые метрики (KPI): Скорость загрузки (Upload speed), время обработки видео (transcoding), количество успешных скачиваний.

3. Пользовательские роли и Сценарии (User Roles & Use Cases)
Описание того, как разные люди будут взаимодействовать с Luminous.

Роли: Зритель (гость), Зарегистрированный пользователь, Контент-мейкер, Администратор.

Основные Use Cases:

Загрузка: Пользователь выбирает файл -> Система проверяет формат -> Идет прогресс-бар -> Видео доступно.

Скачивание: Пользователь выбирает качество (720p, 1080p) -> Система формирует пакет -> Начинается загрузка на устройство.

4. Функциональные требования (FR)
Техническое описание того, что должна «уметь» система.

FR.1 (Загрузка): Поддержка форматов mp4, mov, avi. Ограничение по весу (например, до 2 ГБ).

FR.2 (Обработка): Автоматическое конвертирование видео в несколько разрешений (транскодирование).

FR.3 (Хранение): Использование облачного хранилища (S3-совместимого) с резервным копированием.

FR.4 (Скачивание): Генерация временных токенов для безопасного скачивания файлов.

5. Архитектурная схема (High-Level Design)
Диаграмма, описывающая компоненты:

Frontend: React/Next.js для интерфейса.

Backend: API для обработки запросов (Go или Python хорошо подходят для работы с видео).

Video Processing: Очереди задач (например, RabbitMQ + FFmpeg) для сжатия видео.

Storage: Объектное хранилище (AWS S3, MinIO).

6. Стак технологий (Tech Stack)
Список инструментов:

Язык: Python (Django/FastAPI) или Node.js.

База данных: PostgreSQL (метаданные) + Redis (кеширование).

Инструменты видео: FFmpeg (библиотека №1 для работы с видео).
